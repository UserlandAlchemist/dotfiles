TASK 2: Document vanilla Trixie divergence
───────────────────────────────────────────

**Goal:** Create docs/audacious/VANILLA-DIVERGENCE.md

**Content to document:**

For each area where we diverge from stock Debian Trixie:

## File/Area Modified
**Location:** /path/to/file
**Divergence:** What we changed
**Reason:** Why we changed it
**Vanilla path:** What stock Debian does
**Recovery impact:** How this affects reinstalls

**Known divergences to document:**
- .bashrc edited directly (should use drop-in overlays)
- Systemd units in user/system dirs (custom services)
- Custom scripts in /usr/local/libexec (nas-inhibit, etc.)
- Manual /opt installations (sfizz, zyn-fusion, vcv-rack)
- Custom kernel parameters via UKI
- Power management policies (no suspend on audacious)
- ZFS instead of ext4 for root
- systemd-boot instead of GRUB
- Unified Kernel Images (UKI) instead of separate initrd

**Format:**
- Terse, factual documentation
- No judgment about choices
- Focus on "what differs" and "why"
- Include recovery implications

**When done:**
- Commit with message: "docs: document divergence from vanilla Trixie"
- Use handoff template
